# Project Style Sheet

Use this guide to create a thesis-specific style sheet before large consistency or grammar work. The style sheet should be inferred from the current thesis and updated when the user makes a decision.

## When to Create One

- The thesis is assembled from multiple papers or chapters.
- The user asks for whole-thesis consistency.
- There are repeated questions about terminology, notation, title style, citation style, or bilingual wording.
- A reviewer asks for uniform language or formatting.

## Derivation Rules

- Prefer the thesis's dominant existing convention unless it conflicts with examiner comments, school rules, or clear grammar.
- Record variants before changing them. Do not run blind global replacements.
- Treat technical terms, compound modifiers, proper nouns, and established field phrases separately.
- Keep exceptions explicit so later passes do not "fix" intentional choices.

## Use During Edits

- Before editing, cite the style sheet rule behind each consistency change.
- If no rule exists, add a proposed rule and ask for confirmation when the choice is high-impact.
- After editing, update the style sheet if the preferred form changed.
